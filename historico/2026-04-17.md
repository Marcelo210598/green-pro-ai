# Snapshot - 2026-04-17

## 🎯 O que foi feito hoje

MVP completo do Green Pro AI — do zero ao deploy em produção.

### Estrutura criada
- Next.js 16.2.4 (App Router, Turbopack, TypeScript, Tailwind v4, shadcn/ui base-ui)
- Prisma 6 + Neon PostgreSQL (banco compartilhado com o bot Rei do Green)
- NextAuth v5 com Credentials provider, bcrypt, JWT + campo `plan`
- Stripe stub desativado (PAYMENTS_ENABLED=false), estrutura pronta pra ativar depois

### Páginas
- `/` — Landing (Hero, StatsGrid, ComoFunciona, UltimosResultados, FinalCTA + aviso legal Lei 14.790/2023)
- `/tips` — Tips do dia com gating free/pro (tudo liberado no MVP)
- `/historico` — Tabela pública de resultados passados + stats
- `/planos` — 3 cards: Free / Pro R$49 / Premium R$149 (botões "Em breve" enquanto Stripe desativado)
- `/login` e `/cadastro` — Formulários com validação zod

### APIs
- `GET /api/tips` — tips do dia com status derivado (apostável/informativo/fora-de-mercado)
- `GET /api/stats` — win rate, ROI, greens/reds, banca
- `GET /api/historico` — histórico completo com paginação básica
- `POST /api/register` — cadastro com bcrypt
- `/api/auth/[...nextauth]` — NextAuth handlers
- `/api/stripe/checkout` e `/api/stripe/webhook` — stubs (503 enquanto desativado)

### Fixes aplicados durante build
- Trocado `<Button asChild>` por `<Link className={buttonVariants(...)}>` (base-ui não tem Slot)
- `/login` quebrava no build por `useSearchParams` — extraído `LoginForm.tsx` client dentro de Suspense
- `next.config.ts` com `turbopack.root` pra silenciar warning de multiple lockfiles

## 📝 Decisões técnicas importantes

- **Banco compartilhado com o bot** — site é read-only, bot escreve, site apenas lê
- **Prisma 6** (não 7) — Prisma 7 mudou breaking change na config de `url` do datasource
- **buttonVariants + Link** em vez de `asChild` — mais limpo com base-ui
- **applyGating com paymentsEnabled()** — durante validação libera tudo, gating automático quando Stripe ativo
- **Não rodar `prisma migrate`** — schema já existia no Neon (criado pelo bot), só `prisma generate`

## ⚠️ Problemas encontrados e soluções

- **Prisma 7 breaking change**: `datasource url` não aceita mais `env()` da mesma forma → downgrade pra Prisma 6
- **`useSearchParams` em `/login`**: quebra no build (precisa Suspense) → extraído LoginForm.tsx client
- **`asChild` no Button (base-ui)**: não existe no @base-ui/react/button → substituído por `Link + buttonVariants`
- **Vercel workspace root warning**: múltiplos package-lock.json → `turbopack.root` no next.config.ts
